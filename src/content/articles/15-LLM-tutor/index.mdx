I want to learn Rust.

I asked Claude to interview me first to get to know my background. I have latent experience in C++ and use Python daily. This background is important to optimize the course as not all concepts are foreign. It turns out a lot of concepts in C++ transfer quite easily to rust.
I told Claude what method works best for me, i.e. small projects to apply new concepts.
For shits and giggles, I asked for nvim tips and tricks.

## The curriculum

## Capstone project: [Chip 8 emulator](https://github.com/ThomasVanRiel/chip8-rust)

As a capstone project to solidify the rust concepts I have learned in the process, I opted to write a chip 8 emulator.
I have written such an emulator before in C++ using OpenGL, and for playstation 3, which was a pleasant experience during which I solidified my C++ and platform knowledge.
The emulator in rust is hence both a learning experience and a throwback to my time in Kortrijk, studying [Digital Arts & Entertainment](https://www.howest.be/en/programmes/bachelor/digital-arts-and-entertainment).

### The process

The process looked a lot like a regular interaction with a LLM chatbot, except for one major difference: I didn't do the prompting, Claude Code was prompting me.
I could ask for clarification but in the end, the code is fully written by me with directions from my tutor Claude.

### The emulator

Select a ROM from the dropdown (autogenerated from [this repo](https://github.com/kripod/chip8-roms)) and click the load button. Loading a local ROM file is also possible by clicking the field below the dropdown.  
The 16 key input panel is mapped to `1234`,`qwer`,`asdf`,`zxcv`.

## Claude Code configuration using `CLAUDE.md`

The CLAUDE.md file is used as a persistent memory to provide the LLM enough context to behave like a consistent human tutor rather than a stateless assistant.
The file is fully authored by Claude and is structured in several sections:

- **Student Profile**: Background, learning approach and style, problem solving behaviour, ...
- **Curriculum Plan**: Project by project goals and the current status thereoff to prevent reteaching covered concepts.
- **Session Log**: Chronological summaries of the classes, acting as episodic memory to store the state of the curriculum, e.g. what was introduced, what gaps exist, ... Full class notes are referenced here.
- **Reference Sheets and Class Notes Index**: Quick links to the full class notes and an index of what was learned in what class or what project.
- **Current State and Upcoming Topics**: What Rust concepts do I already know and what should still be thaught? This section is the key to fill Claude's context to continue the class.
- **Notes & Observations**:
- **Tutor Habits**: Do and don'ts and general tutor behaviour.
  The key in this section is in my opinion the instruction: "Let Thomas struggle productively", as reading compiler messages and exploring documentation is a key programming skill that needs to be acquired.

The complete contents of `CLAUDE.md` can be read in my [`learnrust` repository](https://github.com/ThomasVanRiel/learnrust/blob/main/CLAUDE.md).
